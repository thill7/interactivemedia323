The Square Root of Negative 1

Letter by letter, with Pixabay API

This method of posting was logistically difficult, but I feel like it worked at least 80% as intended. It's hard to see what's going on behind the scenes, but the text has been stored as a variable in this page. For each letter of each word, an image search for that letter is made and then added to the page. Since a search is made for each letter, the results should be unique each time the page is loaded.

I chose to do this in order to emphasize a point touched on in the text illustrated: webpages can be a medium for dynamic poetry, however loosely poetry may be defined in this context. As Bok argues, any novel development in poetry is unlikely to occure unless the tools used are reevaluated.

The Square Root of -1

Reddit, JSON, and Imgur

This one, I'll admit, became a lot more covoluted than originally intended. First, I created a subreddit: /r/thesquarerootofnegative1. Then I made a post with the first line of the text, followed by a chain of replies with two separate accounts until the entirety was spelled out. But what you're seeing here isn't that post, but rather all of the information about those posts as its seen by the servers, in a format known as JSON (Javascript Object Notation): basically, lists of lists. This kind of data has numerous applications, and was actually used to create the little applet on my homepage, which loads random images from different subreddits. My purpose in this one is a little more abstract. I wanted to try to show the layers of a webpage that aren't usually visible. The rigging that makes the webpage as a medium truly unique.
